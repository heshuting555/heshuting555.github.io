

本指南将帮助你将本地通过 Docker 修改好的网站部署到 GitHub Pages。

## 1. 准备工作：修改配置文件
在提交代码前，必须确保 `_config.yml` 中的路径与你的 GitHub 仓库一致。

1.  打开 **`_config.yml`**。
2.  修改 **`url`**：
    * 设置为 `https://<你的用户名>.github.io`。
3.  修改 **`baseurl`**：
    * 如果你的仓库名是 `<你的用户名>.github.io`（个人主页项目），请设为 `""`。
    * 如果你的仓库名是 `my-lab`（子项目），请设为 `"/my-lab"`。

---

## 2. 提交代码到 GitHub
在你的本地终端（或 VSCode 终端）执行以下操作：

```bash
# 1. 查看修改状态
git status

# 2. 将所有修改加入暂存区
git add .

# 3. 提交修改
git commit -m "Update: My lab website is ready"

# 4. 推送到 GitHub
git push origin main
```

---

## 3. 在 GitHub 上开启自动部署 (Actions)
`al-folio` 依赖复杂的插件，因此不能使用 GitHub 默认的 Jekyll 构建引擎，必须使用 **GitHub Actions**。

1.  登录 GitHub，进入你的仓库。
2.  点击顶部菜单栏的 **Settings**（设置）。
3.  在左侧侧边栏点击 **Pages**。
4.  在 **Build and deployment** > **Source** 下，下拉选择 **GitHub Actions**。



---

## 4. 监控部署进度
1.  点击仓库顶部的 **Actions** 选项卡。
2.  你会看到一个名为 `deploy` 或 `pages build and deployment` 的工作流正在运行。
3.  **等待图标变绿**（通常需要 2-3 分钟）。
4.  运行成功后，刷新 **Settings -> Pages** 页面，你会看到你的网站链接已生成。

---

## 5. 常见问题排查 (FAQ)

### Q1: 部署成功了，但网页是一片空白或者样式乱了？
* **原因**：通常是 `_config.yml` 里的 `baseurl` 设置错了。
* **检查**：如果你的网址是 `.../al-folio/`，那么 `baseurl` 必须是 `/al-folio`（注意开头的斜杠，结尾不要斜杠）。

### Q2: 为什么 Actions 报错了？
* **原因**：可能是某个插件版本冲突或 `Gemfile.lock` 的问题。
* **解决**：
    1.  点击 Actions 里报错的步骤查看日志。
    2.  确保你本地运行 `bundle install` 产生的 `Gemfile.lock` 已经提交到了 GitHub。

### Q3: 以后如何更新内容？
* **流程**：本地修改文件 -> Docker 预览满意 -> `git add .` -> `git commit` -> `git push`。GitHub 会自动感应并重新发布，你不需要做额外操作。
